# Stage 4: PCAP Parsing and Traffic Profile Extraction

Stage 4 processes PCAPs produced by previous stages and generates structured traffic/application profiles that are written back to the database.

Previous stage: [stage3/README.md](../stage3/README.md)  
Next stage: [stage5/README.md](../stage5/README.md)

## What Stage 4 Does

- Reads PCAP files from `stage4/pcap/`.
- Runs CICFlowMeter (via Docker) to produce flow-level CSV features.
- Runs `tshark` field extraction and additional dataframe processing.
- Maps ports to IANA service names.
- Uses LLM-assisted post-processing to infer application-level traffic parameters.
- Updates DB records (`traffic_parameters` or `agent_traffic_parameters`) with extracted payloads.

## Main Files

- [process_pcap.py](process_pcap.py): processes agent-generated PCAP entries (`agent_traffic_parameters`).
- [process_pcap_traffic_parameters.py](process_pcap_traffic_parameters.py): processes legacy/standard traffic parameter rows.
- [src/flowmeter_setup.py](src/flowmeter_setup.py): prepares and manages CICFlowMeter container runtime.
- [src/process_traffic_profile.py](src/process_traffic_profile.py): LLM-assisted traffic profile processing.
- [src/extract_hostname.py](src/extract_hostname.py): hostname extraction from traffic metadata.
- [src/iana_port_mapping.csv](src/iana_port_mapping.csv): protocol/port mapping source.

## Prerequisites

- Python >= 3.13
- Docker Engine (for flowmeter container)
- `tshark` installed
- PostgreSQL configured

## Environment Variables

- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `POSTGRES_DB`
- `POSTGRES_CONTAINER`
- `POSTGRES_PORT`

LLM/API variables may be required depending on your configured model in `process_traffic_profile`.

## Install

From `stage4/`:

```bash
uv sync
```

## Run

Agent traffic parameter path:

```bash
uv run python process_pcap.py
```

Standard traffic parameter path:

```bash
uv run python process_pcap_traffic_parameters.py
```

Both scripts run continuous polling loops.

## Outputs

- Derived traffic profiles written to PostgreSQL.
- Intermediate CSV artifacts in stage-local output folders (for CICFlowMeter/tshark processing).
- Processing logs under `stage4/logs/`.

## Notes

- Missing or corrupted CSV/PCAP artifacts are handled with fallback logic and skip markers.
- Stage 4 expects PCAP naming conventions generated by upstream stages.
